# UFW 防火墙优化脚本

限制指定国家/地区 IP 访问的 UFW 防火墙规则管理工具。

## 功能特性

- 支持 8 个国家/地区选择 (中国、印度、俄罗斯、美国、日本、韩国、巴西、印度尼西亚)
- TUI 交互界面 (whiptail)
- 自动检测已有规则，智能处理
- 规则备份与恢复
- 错误自动回滚
- 清理旧的 "任意访问" 规则

## 工作原理

```
执行流程:
1. 用户选择区域 → 2. 备份当前规则 → 3. 分析现有规则
       ↓
4. 添加新规则 (仅允许选定区域IP) → 5. 清理旧规则 → 6. 设置 deny incoming
       ↓
7. 完成
```

### 智能规则处理

| 场景 | 处理方式 |
|------|----------|
| 内网 IP 规则 | 自动跳过，不重复限制 |
| 外网特定 IP 规则 | 提示用户，跳过该端口 |
| 任意访问 (Anywhere) 规则 | 清理并替换为区域限制 |
| SSH 连接 | 新规则生效后再清理旧规则，确保不断连 |

## 使用示例

### 1. 运行脚本

```bash
sudo ./banIP.sh
```

### 2. 主菜单

```
┌─────────────────────────────────────┐
│  主菜单                              │
│                                      │
│  1. 设置防火墙规则                   │
│  2. 恢复原有规则                     │
│  3. 查看当前 UFW 状态                │
│  4. 退出                             │
└─────────────────────────────────────┘
```

### 3. 设置防火墙规则

```
1. 选择 "设置防火墙规则"
2. 选择要限制的区域 (如: 日本)
3. 等待备份完成
4. 等待添加规则 (进度条显示)
5. 完成
```

### 4. 恢复规则

如果配置后出现问题，选择 "恢复原有规则" 即可恢复到备份状态。

## 运行时文件

脚本运行时会自动生成以下文件：

| 文件 | 说明 |
|------|------|
| `restricted_ports.txt` | 从当前 UFW 规则提取的端口 |
| `china_ips.txt` | 选定区域的 IP 段列表 |
| `ufw.policy` | UFW 规则备份 |

这些文件可以在运行后删除，不影响再次运行。

## 安全机制

### 备份保护
- 执行配置前自动备份
- 备份失败则阻止继续

### 断连保护
- 先添加新规则，确保 SSH 可访问
- 再清理旧规则
- 最后设置 deny incoming

### 错误回滚
- 添加规则失败时自动回滚
- 删除本次添加的规则
- 尝试恢复到备份状态

## 注意事项

- 脚本需要 `sudo` 权限
- 运行前请确保了解防火墙配置
- 误配置可能导致无法远程访问
- 建议首次在测试环境运行
- 确保已备份重要规则
